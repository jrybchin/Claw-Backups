# Response Logic Index

## Folder purpose
Rules for how Claw should respond, structure answers, and adapt communication style.

## Table of contents

### 1. Response Logic
**Purpose:** Main rules for response structure, brevity, adaptation, and readability.

**Sections:**
- **Purpose** — defines the goal of the response system.
- **Core Principles** — explains the main communication values.
- **Response Modes** — defines the main types of responses Claw should use.
- **Trigger Logic** — explains when to stay short, detailed, or more output-focused.
- **Standard Response Shape** — defines when summary/conclusion/next-steps may help.
- **Length and Density Rules** — explains how response length should be controlled.
- **Conversation Style Rules** — explains the intended tone and readability.
- **Information Disclosure Rules** — explains what implementation/storage details to omit unless needed.
- **Adaptation Rules** — explains how style should evolve based on the user.
- **Momentum Preservation Rules** — explains how responses should avoid slowing workflow.
- **Task Completion Rules** — explains that dependent steps should be finished before reporting completion.
- **Example Use Rule** — limits unnecessary examples.
- **Future Expansion Note** — reserves room for later school/assignment behavior.
- **Relationship To Other Docs** — explains where this doc fits in the larger system.
- **Status** — marks the doc as an evolving response system.

## Notes
- This index only covers documents in the same folder.
- It does not describe subfolders or documents inside subfolders.
