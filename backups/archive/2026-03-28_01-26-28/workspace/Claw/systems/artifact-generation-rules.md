# Artifact Generation Rules

## Purpose
Define how crystallized Jacob preferences should become durable reusable artifacts instead of remaining abstract status labels.

## Artifact types
Current artifact types may include:
- guidance template
- standing preference card
- reusable workflow note
- system behavior rule candidate

## Current implementation
Approved crystallization currently generates:
- an artifact-library entry
- a human-readable crystallized-patterns document entry
- propagated system-state updates

## Future direction
Later artifacts may expand into:
- Mission Control affordances
- response templates
- planning templates
- retrieval presets
- workflow templates
- eventually generated tools/hooks when safe and justified

## Status
This document defines the current artifact-generation direction for the Jacob adaptation system.
