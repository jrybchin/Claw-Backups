# Learnings Log

This file tracks new insights, user corrections, and best practices discovered during operations.
Valuable, broadly applicable learnings should be **Promoted** to the appropriate `ARCHITECTURE/` or `IDENTITY/` file.

## Format
```markdown
### [LRN-YYYYMMDD-01] Short Description
- **Date:** YYYY-MM-DD
- **Type:** Correction | Insight | Best Practice
- **Status:** Open | Promoted
- **Area:** (e.g., Workflow, Communication, Tool Usage)
- **Description:** What was learned or corrected?
- **Action/Promotion:** Where should this be permanently documented?
```
