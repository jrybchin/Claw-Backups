# Human System Explanation

## What this folder is for
This folder explains how the Claw system works at a big-picture level.

The goal is to make it easier for a human to:
- understand what the system is doing
- understand why it is structured the way it is
- debug it more easily
- explain it to other people
- present it professionally
- learn from it and build on it

## What Claw is trying to become
Claw is being built as more than a normal chatbot.

The goal is to turn it into a structured AI system that can help with:
- productivity
- life organization
- email and planning
- school and studying
- feature building and automation
- documentation and recovery
- long-term project growth

## Why documentation matters so much
A big part of the system is writing things down clearly.

That matters because:
- chat history is messy
- context can be lost
- features get more complicated over time
- crashes and failures happen
- future versions of the system need to understand what was already built
- the roadmap can drift if it is not actively kept aligned with the real system

So the docs are not just notes.
They are part of how the system thinks and stays organized.

## How the docs are organized
### Top-level Claw coordination docs
Some important coordination docs may live at the top level of the Claw Drive structure or as top-level reference points in the broader system.

Examples:
- Claw Plan
- Active Tasks List
- Claw Full Sweep Prompt

### Features
Feature docs explain specific things being built.

Examples:
- Recovery System
- GOG integration

### Systems
System docs cover broader rules that affect many features.

Examples:
- documentation rules
- memory and source-handling rules
- document role classification

### Logic
Logic docs explain the overall structure of the system.

This includes:
- the machine-facing logic doc
- the human-facing explanation doc
- response logic
- maintenance prompts
- logic indexes

## Why there are two kinds of logic docs
There are two different needs:

### 1. A doc for other AIs
This version is meant to:
- be copied into other systems
- help other AIs understand the logic
- preserve the deeper structure of how Claw works
- support migration and collaboration

### 2. A doc for humans
This version is meant to:
- explain things simply
- help with debugging
- help with presenting the project
- help the user learn how the system works

These should stay aligned, but they do not need to sound the same.

## How Claw tries to work well
The main ideas are:
- keep things organized
- avoid repeated loops
- protect the user’s time
- search broadly before assuming something is missing
- keep enough raw information to stay accurate
- write down important rules and lessons
- recover cleanly when something breaks
- make future debugging easier
- respond in a way that is easier to skim and finish reading
- keep the roadmap and the logic layer aligned with the actual system state

## How recovery fits into the system
The system is being built so that if something breaks:
- the current state is easier to understand
- the important docs already exist
- recovery can happen faster
- backup systems like Codex can help with less confusion

That is why the Recovery System feature exists.

## Why response logic matters too
The system is also being shaped so responses are:
- easier to scan
- less overwhelming
- more conversational
- more adaptive to what the user actually pays attention to

That matters because a strong system is not just smart internally.
It also needs to communicate in a way the user will actually use.

## What is most important right now
The system is currently in a foundation and architecture phase.

That means the biggest priorities are:
- keeping the system stable and coherent
- strengthening documentation and continuity
- improving recovery and backup workflows
- improving memory and source handling
- keeping the roadmap and central logic aligned with everything else

## Why this matters beyond just using the bot
This project is also meant to help the user:
- develop technical understanding
- learn how AI systems are structured
- build something impressive and practical
- talk about it professionally
- show it to employers or recruiters
- become better at designing and using AI tools
