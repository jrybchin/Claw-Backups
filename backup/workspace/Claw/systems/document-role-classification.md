# Document Role Classification

## Purpose
Classify current Claw documents by role so update behavior, review habits, and documentation expectations become clearer.

## Role Types
- **Live** — should stay current and may update often during active system work
- **Dynamic** — should actively shape behavior in the background and be treated as live operating logic, even if not rewritten every turn
- **Acute** — important current explanatory logic that may be revised intentionally but is not as constantly active as the dynamic layer
- **Static** — preserved systems or instructions that are important when invoked, but not constantly active in the background
- **Reference** — historical, repair, or mostly stable information
- **Navigation** — helps humans find documents quickly

## Interpretation Notes
- **Dynamic** docs should always be influencing system behavior in the background.
- **Dynamic** does not mean they must be rewritten constantly; it means they should be actively used.
- **Live** docs are the main working records that should be updated often during active system-building work.
- **Static** docs matter when invoked, but are not part of the always-running background logic.
- **Navigation** docs are primarily human-facing and should only change when structure or contents make an update necessary.
