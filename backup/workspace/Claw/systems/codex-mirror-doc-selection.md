# Codex Mirror Doc Selection

## Purpose
Define which Claw docs should be mirrored into the Windows Codex-facing mirror folder.

## Mirror Principles
Mirror only the most important docs for:
- central alignment
- current state
- recovery
- core system understanding
- high-value continuity

Do not mirror every Claw doc by default.

## Current Mirror Set

### Core
- Claw Plan
- Claw Full Sweep Prompt
- Transplantable System Logic
- Human System Explanation
- Documentation Rules
- Memory & Source Handling System
- Active Tasks List

### Recovery
- Live System Status Report
- Crash Prevention & Fail-Safe Checklist
- Backup Lifeline: Codex Recovery Workflow
- Codex Recovery Agent Instructions

### Reference
- GOG Feature Overview
- GOG Setup & Repair Log

## Future Addition Rule
Consider adding a doc to the Codex mirror when it becomes:
- a central alignment doc
- a dynamic/background logic doc
- a live status / active queue doc
- a high-value recovery/debugging doc
- a key continuity reference Codex is likely to need

## Immediate Sync Rule
After a high-value change, trigger an immediate Codex mirror sync.

High-value changes include:
- Claw Plan updates
- dynamic logic updates
- live status updates
- recovery doc updates
- active task changes
- major structural or continuity-related changes

## Status
This mirror set should evolve carefully rather than expanding into a full raw clone of the entire Claw folder.
